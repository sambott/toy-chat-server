Chat Client Hackathon

Your challenge is to create a chat client.

  • Form groups of 2-3
  • Decide on a platform and technology stack
    • You can use any platform you like
      • Web based
      • Console app
      • Mobile app
      • Windows/MacOS Client
      • Even an app for a games console ... if you happen to have brought one
    • We'll provide a couple of starter projects (below) to demonstrate the API or to help get you started if you would like.
    • Nobody is expecting a fully working product.
      • Pick a technology you think will be fun, and spend the time aiming for basic functionality.
      • Or pick a technology you are comfortable with and spend time adding some polish or interesting features (perhaps nice look/feel, or emojis, or room-switching....)
  • Hack! You'll have four hours to build something
  • We'll be wondering around. Ask if you would like any assistance.
  • At the end of the afternoon, we'll ask you to show us what you came up with.
    • What technology stack did you decide on? Why?
    • What did you get working? What didn't you?
    • Any interesting bits of code you'd like to show off??
    • Demonstrate your client.

API

{{backendProtocol}}{{backendRoot}}

Routes
GET/roomsRetrieves a list of the rooms with recent activity.
GET/rooms/:roomGets most recent messages for the chat room.
POST/rooms/:roomPost a message to the chat room
GET/rooms/:room/wsOpen a WebSocket for event-driven sending and receiving of messages
Objects

A sent message should resemble:

    {
      "user": "your name",
      "message": "the message to send"
    }

Messages you receive will resemble:

    {
      "user":"their name",
      "message":"message text",
      "dateTime":1463930717000, // milliseconds since unix epoch
      "room":"room_name"
    }
